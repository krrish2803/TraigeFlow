# TriageFlow - Session Summary

## What was built

### Auth System
- **`src/lib/auth.ts`** вЂ” NextAuth v5 config with GitHub + Credentials provider (demo login)
- **`src/app/api/auth/[...nextauth]/route.ts`** вЂ” Route handler for auth
- **`src/app/login/page.tsx`** вЂ” Login page with GitHub OAuth and demo email login

### Data API Routes
All read-only API routes backed by in-memory Lemma store with auto-seed:
- **`src/app/api/signals/route.ts`** + **`[id]/route.ts`** вЂ” GET signals list / single signal
- **`src/app/api/clusters/route.ts`** вЂ” GET clusters
- **`src/app/api/drafts/route.ts`** вЂ” GET drafts
- **`src/app/api/releases/route.ts`** вЂ” GET release impact data (risk score, blocking/caution/safe counts, last digest)
- **`src/app/api/dashboard/route.ts`** вЂ” GET aggregate metrics + recent activity
- **`src/app/api/activity/route.ts`** вЂ” GET activity log

### Data Access Layer
- **`src/lib/api-data.ts`** вЂ” Shared server-side data access. Auto-creates tables + seeds from `lib/seed-data.ts` (15 signals, 3 clusters, 3 drafts, 5 activity entries)
- **`src/lib/use-data.ts`** вЂ” Client-side React hooks (`useSignals`, `useSignal`, `useClusters`, `useDrafts`, `useActivity`, `useReleases`, `useDashboard`) for data fetching

### Frontend Updates
Pages updated to fetch from real API routes instead of hardcoded demo data:
- **`src/app/(app)/inbox/page.tsx`** вЂ” Signal list from API, filterable by source
- **`src/app/(app)/clusters/page.tsx`** вЂ” Cluster list from API
- **`src/app/(app)/drafts/page.tsx`** вЂ” Draft list with pending/approved filters
- **`src/app/(app)/releases/page.tsx`** вЂ” Release risk dashboard with real cluster/draft data
- **`src/app/(app)/dashboard/page.tsx`** вЂ” Summary metrics with trend chart (recharts)

### Components
- **`src/components/signals/SignalDetailPanel.tsx`** вЂ” Updated to fetch from API (`useSignal` hook)
- **`src/components/activity/ActivityFeed.tsx`** вЂ” New component for activity log display

## Key Design Decisions
- In-memory Lemma store (data resets on server restart, auto-seeded on first request)
- API routes are all static (pre-rendered) except `[...nextauth]` and `[id]` params
- Auth is configured but pages are NOT currently protected by middleware
- Recharts used for dashboard trend chart
